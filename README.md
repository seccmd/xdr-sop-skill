## 二、核心解决的问题（痛点明确，贴合平台审核）
- 供应链投毒应急无统一流程，临场拼凑步骤、易漏项、留隐患
- 多安全设备分散，处置时频繁切换后台，操作繁琐、效率低下
- 供应链攻击隐蔽性强，人工排查易踩坑（如漏清CI缓存、未重置凭证）
- 团队应急标准不统一，应急质量依赖个人经验，新人上手难度大

## 五、适配场景
1.  开源组件投毒应急（npm/pip/gem等包投毒、依赖污染）
2.  CI/CD环境恶意感染、恶意文件传播处置
3.  多设备统一应急操作，减少平台切换开销
4.  团队应急流程标准化培训、新人上手指导
5.  供应链安全应急演练与复盘
